## Общее описание
Данный проект содержит решения для трех заданий по работе с витринами данных:

Задание 2.1: Устранение дубликатов в витрине dm.client

Задание 2.2: Восстановление недостающих данных в витрине dm.loan_holiday_info

Задание 2.3: Исправление некорректных балансов счетов в витрине dm.account_balance_turnover

Задание 2.1: Устранение дубликатов в витрине клиентов
Описание проблемы
Витрина dm.client содержит информацию по клиентам банка. В ходе проверки обнаружено наличие дублирующихся записей, которые необходимо устранить.

### Структура витрины
Составной ключ для идентификации уникальной строки:

client_rk – уникальный код клиента

effective_from_date – дата начала действия записи

## Задание 2.2: Восстановление данных витрины кредитных каникул

### Описание проблемы

Витрина `dm.loan_holiday_info` содержит информацию по кредитным каникулам, сделкам и продуктам. В ходе анализа выявлено, что данные присутствуют только для трех дат:

- 2023-01-01 (3000 записей)
- 2023-03-15 (3500 записей)
- 2023-08-11 (3502 записи)

Остальные 362 дня 2023 года отсутствуют в витрине.

### Источники данных

Витрина формируется из трех таблиц:
- `rd.deal_info` – информация по сделкам клиентов
- `rd.loan_holiday` – информация о кредитных каникулах
- `rd.product` – информация о продуктах банка

### Решение

#### Файлы решения

| Файл                            | Назначение |
|---------------------------------|------------|
| `configurations.py`             | Конфигурация подключения к БД и параметры загрузки |
| `2_2.py`                        | Основной скрипт для загрузки данных и обновления витрины |
| `refresh_loan_holiday_info.sql` | SQL-процедура для пересчета витрины |

#### Функциональность скрипта

1. **Анализ текущего состояния** – определение диапазона дат и количества пропущенных дат в витрине
2. **Загрузка данных из CSV** – загрузка файлов `deal_info.csv` и `product_info.csv` в RD слой с автоматическим определением кодировки и формата дат
3. **Удаление дубликатов** – очистка данных перед загрузкой
4. **Создание процедуры** – создание SQL-процедуры `refresh_loan_holiday_info()`
5. **Пересчет витрины** – полная перезагрузка витрины по прототипу

#### Результат

После выполнения скрипта витрина `dm.loan_holiday_info` будет полностью пересчитана на основе актуальных данных из CSV файлов.

## Задание 2.3: Исправление балансов счетов

### Описание проблемы

Витрина `dm.account_balance_turnover` отражает изменение баланса счетов по дням. В ходе проверки обнаружено, что у счетов иногда отличается:

- `account_out_sum` – сумма на конец дня
- `account_in_sum` – сумма на начало следующего дня

Эти значения должны быть равны, но в данных встречаются расхождения.

### Источники данных

Витрина формируется из трех таблиц:
- `rd.account` – информация по счетам клиентов
- `rd.account_balance` – суммы на начало и конец дня по счетам
- `dm.dict_currency` – справочник валют

### Решение

#### Файлы решения

| Файл                                   | Назначение |
|----------------------------------------|------------|
| `configurations.py`                    | Конфигурация подключения к БД |
| `task_2_3_solution.py`                 | Основной скрипт для исправления данных и обновления витрины |
| `task_2_3_correct_account_in_sum.sql`  | Запрос для определения корректного `account_in_sum` |
| `task_2_3_correct_account_out_sum.sql` | Запрос для определения корректного `account_out_sum` |

#### Функциональность скрипта

1. **Анализ несоответствий** – поиск и подсчет расхождений между `account_in_sum` и `account_out_sum` соседних дней
2. **Определение корректных значений**:
   - Если `account_in_sum` текущего дня отличается от `account_out_sum` предыдущего дня, корректным считается `account_out_sum` предыдущего дня
   - Если `account_out_sum` текущего дня отличается от `account_in_sum` следующего дня, корректным считается `account_in_sum` следующего дня
3. **Исправление данных** – обновление таблицы `rd.account_balance`
4. **Создание процедуры** – создание SQL-процедуры `refresh_account_balance_turnover()`
5. **Перезагрузка витрины** – полная перезагрузка витрины с применением коррекции
6. **Проверка результата** – верификация отсутствия несоответствий

#### Результат

После выполнения скрипта:
- Данные в таблице `rd.account_balance` будут исправлены
- Витрина `dm.account_balance_turnover` будет перезагружена с корректными значениями
- Все несоответствия между `account_in_sum` и `account_out_sum` будут устранены
